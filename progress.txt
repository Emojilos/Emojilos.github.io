## Completed this session (continued 4)
- TASK-024 (Server hit detection: raycast, damage, kills)
  - Created packages/server/src/physics/hitDetection.ts:
    - rayAABB(): slab-method ray-AABB intersection, returns distance or null
    - raySphere(): analytic ray-sphere intersection for headshot detection
    - performHitDetection(): iterates all alive enemy players, tests head sphere (HEAD_RADIUS=0.15 at HEAD_OFFSET_Y=1.6) then body AABB (PLAYER_RADIUS × PLAYER_HEIGHT), returns closest hit with isHeadshot flag
    - Skips self and teammates (no friendly fire)
    - HitTarget interface: {sessionId, x, y, z}
    - HitResult interface: {targetId, isHeadshot, distance}
  - Updated packages/server/src/rooms/GameRoom.ts shoot handler:
    - Server-authoritative origin: player position + EYE_HEIGHT (ignores client origin)
    - Validates and normalizes client direction vector (rejects NaN, zero-length)
    - Builds targets list from alive players in room
    - Calls performHitDetection() with weapon range as max distance
    - On hit: damage = config.damage.head or config.damage.body (weapons.ts already has both)
    - victim.hp -= damage, sends 'hit' event to shooter (HitEvent with targetId, damage, isHeadshot, direction)
    - Sends 'damaged' event to victim (damage, isHeadshot, attackerId, reversed direction for damage indicator)
    - On kill (hp <= 0): victim.isAlive=false, killer.kills++, victim.deaths++
    - Creates KillEventSchema and pushes to state.killFeed (synced to all clients via Colyseus)
    - Broadcasts 'kill' event (KillEvent with killerName, victimName, weaponId, isHeadshot)
    - Console logs kills with weapon and headshot indicator

- Input validation: MAX_DELTA_TIME = 0.25s, rejects non-finite, negative, wrong types
- Server CollisionWorld in packages/server/src/physics/ — mirrors client version without Three.js
- lastProcessedSeq (uint32) in PlayerSchema — clients will use for prediction reconciliation
- velocityY, isGrounded, lastShootTime are server-only fields (not @type decorated, not replicated)
- @colyseus/schema v2.0.37 uses legacy decorators (experimentalDecorators: true)
- tsx needs --tsconfig flag to respect experimentalDecorators from tsconfig.json
- Room code uses ABCDEFGHJKLMNPQRSTUVWXYZ23456789 charset (no 0/O/1/I confusion)
- Room code is exposed as metadata for client discovery via getAvailableRooms()
- Nickname regex: /^[A-Za-z0-9_]+$/, length 3-16
- PlayerSchema uses flat float32 for position/rotation (not nested Schema) for bandwidth efficiency
- GameState.players is MapSchema<PlayerSchema> keyed by sessionId
- GameState.killFeed is ArraySchema<KillEventSchema>
- PLAYER_SPEED = 250 u/s, GRAVITY = -20, JUMP_VELOCITY = 8, EYE_HEIGHT = 1.65m
- PLAYER_RADIUS = 0.3, PLAYER_HEIGHT = 1.8 (collision capsule approximated as AABB)
- applyMovement() is a pure function — returns new state, doesn't mutate input. Ready for server use.
- CollisionWorld.resolve() runs up to 4 iterations to handle corner sliding
- Mouse sensitivity = 0.002, Euler order YXZ for proper FPS camera
- Colyseus 0.15.57 on server, colyseus.js ^0.15.0 on client
- Scene uses PCFSoftShadowMap, fog (30-60 range), directional light shadow camera covers 25 units
- Weapon overlay uses separate scene+camera with autoClear=false, clearDepth before render
- Deagle recoil: 120ms duration, sin-based kick curve, 0.04 up / 0.06 back / 0.15 rad rotation
- App class: setState('playing') starts game loop, setState('menu') stops it and disposes 3D
- App exposed as window.app for console testing: app.setState('playing'), app.setState('menu')
- Screen containers in #ui-root: menu-screen, lobby-screen, playing-screen, match-end-screen
- Canvas is display:none except when in PLAYING state
- RemotePlayerManager uses cylinder+sphere (body+head) per remote player, team-colored
- Input sending: every frame when pointer locked, InputMessage with seq/keys/yaw/pitch/deltaTime
- Remote player sync: onStateChange reads state.players MapSchema, updates capsule positions
- onPlayerAdd/onPlayerRemove handle dynamic spawn/despawn of capsule meshes
- LobbyScreen polls Colyseus state every 200ms via setInterval (not event-driven)
- LobbyScreen auto-transitions to PLAYING when server status !== 'lobby'
- updateSettings rebuilds CollisionWorld on map change
- startGame validates team staffing: 1 per team for 1v1, 2 per team for 2v2
- Admin reassignment on leave already handled in GameRoom.onLeave()
- ShootingSystem: raycaster.far set to weapon range, PointLight for muzzle flash, LineBasicMaterial for tracers
- ShootingSystem spread: base + moving (overrides base) + sustained * consecutiveShots; resets after 300ms idle
- Decals use ring buffer (MAX_DECALS=64) to prevent memory growth; CircleGeometry oriented via lookAt(point+normal)
- ShootMessage sent to server: {seq, timestamp, origin: Vec3, direction: Vec3}
- Server 'shoot' handler: validates alive, !isReloading, ammo>0, fire rate via lastShootTime, decrements ammo
- Server 'reload' handler: validates !isReloading, ammo<magazine, sets isReloading=true, clock.setTimeout completes reload
- Client reload: auto-reload at 0 ammo, R key for manual, blocks firing, syncs from server state
- Reload timer stored in seconds (reloadTime ms / 1000), ticks down each frame in update()
- HUD reload indicator: "RELOADING..." with CSS pulse animation, shown/hidden via isReloading flag
- GameHUD reads hp/score/timer from Colyseus state with fallback defaults
- GameHUD.update() called every frame in animate loop
- HUD container appended to playing-screen div, z-index: 10
- Crosshair is pure CSS (no canvas/WebGL), green #00ff44, 20px size
- WeaponId in HUD now reads from server state (currentWeapon on PlayerSchema)
- ShootingSystem.switchWeapon() handles weapon changes: updates ammo, magazine, reloadTime, raycaster range
- WeaponSelectScreen overlay: z-index:20 (above HUD z-index:10), shown during weapon_select phase only
- ClientPrediction: ring buffer of 256 max PendingInput entries, reconcile on server ack via lastProcessedSeq
- Reconciliation: discard confirmed inputs, snap to server pos, replay unconfirmed through applyMovement+collisions
- Teleport threshold: >2 units = instant snap; <2 units = smooth lerp at 20% per frame via correction offset
- Prediction buffer cleared on countdown event (respawn) and on stopGameLoop
- FPSController exposes getPhysicsState/setPhysicsState/applyPositionDelta for reconciliation
- Hit detection: server-authoritative origin (player.x/y/z + EYE_HEIGHT), client direction (validated+normalized)
- Ray-AABB slab method and ray-sphere for headshot detection in packages/server/src/physics/hitDetection.ts
- Head hitbox: sphere at (x, y+1.6, z) radius 0.15; Body hitbox: AABB from feet to PLAYER_HEIGHT
- Weapon damage from config.damage.head / config.damage.body (already separated in weapons.ts)
- Server sends 'hit' to shooter, 'damaged' to victim, 'kill' broadcast to all; killFeed via Colyseus state sync
- No friendly fire: hit detection skips players on same team
- Entity interpolation: InterpolationBuffer stores snapshots per remote player, renders at 100ms delay (2 ticks at 20Hz)
- InterpolationBuffer: ring buffer of ~22 snapshots, lerp position, lerpAngle for yaw (shortest path wrap-around)
- RemotePlayerManager.pushSnapshot() replaces old updatePlayer(), updateInterpolation() called every frame
- Interpolation buffers cleared on countdown event alongside prediction buffers
- WeaponModel now supports all 3 weapons with switchWeapon() — clears old geometry, builds new, resets state
- WeaponModel WEAPON_PARAMS: per-weapon restPosition, recoilDuration/KickBack/KickUp/Rotation
- SSG-08: long barrel, scope (tube+lens rings), bolt handle+knob, wooden stock, muzzle brake; recoil 0.25s
- MP9: compact SMG, stick magazine, folding stock, polymer grip; snappy 0.06s recoil
- tryFire() now handles automatic (config.automatic=true → hold-to-fire) vs semi-auto (fresh click only)
- Weapon switch: 1/2/3 keys (consumeWeaponSlot) + scroll wheel (consumeWeaponScroll) in InputManager
- switchToWeapon() in App: syncs WeaponModel + ShootingSystem + sends selectWeapon to server
- Scroll wheel cycles through WEAPON_IDS cyclically (wraps around)
- KillFeed: max 5 entries, 5s display + 0.5s CSS fade-out, entries auto-removed after fade
- Kill events arrive via broadcast 'kill' message (KillEvent type from shared); also stored in state.killFeed ArraySchema
- Scoreboard: Tab-hold overlay with two team columns, player rows (nickname, K, D), sorted by kills
- Scoreboard data read from Colyseus state.players (nickname, team, kills, deaths, isAlive)
- Tab key tracked as continuous state (tabHeld) in InputManager, with e.preventDefault() to avoid browser tab-focus

## Completed this session (continued 15)
- TASK-025 (Lag compensation: snapshot buffer and rewind)
  - Created packages/server/src/physics/SnapshotBuffer.ts:
    - Ring buffer of WorldSnapshot entries (player positions per tick)
    - Each snapshot stores timestamp + Map<sessionId, PlayerSnapshot> with x/y/z/isAlive
    - BUFFER_SIZE = ceil(TICK_RATE * 200/1000) + 2 (~6 at 20Hz) for ~1 second history
    - record(): stores player positions at current tick timestamp
    - getSnapshotAt(targetTimestamp): finds closest snapshot to target time
    - Clamps rewind to MAX_REWIND_MS (200ms) — older timestamps get clamped to 200ms ago
    - clear(): resets buffer on round/match reset
  - Updated packages/server/src/rooms/GameRoom.ts:
    - SnapshotBuffer instantiated as class member
    - onTick(): records snapshot each tick during gameplay (roundSystem.isPlayable())
    - Shoot handler: reads client timestamp from ShootMessage, calls snapshotBuffer.getSnapshotAt()
    - If snapshot found, builds target list with rewound positions; else falls back to current positions
    - Rewound targets that were alive at snapshot time use historical x/y/z for hit detection
    - Positions are NOT mutated — rewind is read-only (targets array built from snapshot data)
    - snapshotBuffer.clear() called on returnToLobby
  - No client changes needed: ShootMessage already includes timestamp field
  - TypeScript compiles cleanly for server, client, and shared packages
